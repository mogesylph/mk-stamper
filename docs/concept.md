# mk-stamper concept

mk-stamper は、YouTube 動画を再生しながらタイムスタンプと曲情報を記録するための軽量な単一画面アプリです。

ライブ配信、メドレー、長時間動画などを見ながら、曲の開始・終了時刻、曲名、歌手名を素早く記録し、あとでコピーできる形に整えることを目的にしています。

## 大事にすること

### 動画を最優先にする

このアプリで一番重要なのは YouTube プレイヤーです。

入力欄やボタンは、動画を見る作業を邪魔しない範囲でコンパクトに配置します。レイアウトを変更するときは、まず動画領域が小さくなっていないか、縦横比が崩れていないかを確認します。

### 単一画面で完結する

記録、編集、コピー、リセットまでを 1 画面で完結させます。

ページ遷移や複雑な状態管理は持ち込まず、作業中の視線移動と操作回数を少なくします。

### シンプルに保つ

mk-stamper は小さな道具として扱います。

便利そうな抽象化や将来のための構造は、今必要になるまで入れません。機能追加よりも、読みやすく壊れにくい状態を保つことを優先します。

## 主な機能

- YouTube URL または videoId から動画を読み込む
- 現在の再生位置を開始時刻・終了時刻として記録する
- 曲名と歌手名を登録する
- 登録済みレコードを一覧上で編集する
- 開始時刻順に並べ替える
- 全レコードを TSV 形式でコピーする
- 曲マスタにない曲を未登録曲として確認する
- 未登録曲だけをコピーする
- レコードと再生位置をブラウザの localStorage に保存する

## データの扱い

アプリのデータは基本的にブラウザ内に保存します。

- 登録済みレコード: `localStorage`
- 動画の復元情報: `localStorage`
- 曲マスタ: `app/public/songlist.tsv`

サーバーにデータを送信する仕組みは持ちません。

## UI 方針

- YouTube プレイヤーをできるだけ大きく表示する
- 入力欄と操作ボタンはできるだけ 1 行に収める
- 余白や装飾は最小限にする
- スマートフォンでは動画の下に操作 UI を回す
- テーブルは一覧性を優先し、編集操作をその場で完結させる

## 実装方針

フロントエンドは Vue と Vite で構成します。

状態は画面内に閉じ、必要な範囲だけ composable や service に分けます。コンポーネント分割は、責務が明確に分かれる場合に限ります。

このプロジェクトでは、一般的にきれいな設計よりも、今の規模で読みやすく保守しやすい設計を優先します。
